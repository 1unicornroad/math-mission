This skill guides creation of distinctive, production-grade frontend interfaces for a **kid-friendly arcade space shooter** where players practice multiplication by blasting the correct answers before danger reaches them.

The visual direction should be inspired by **clean low-poly isometric sci-fi models**: modular white structures, warm orange accents, soft gray mechanical forms, dusty planetary terrain, compact ships, and toy-like industrial details. The overall feel should be **playful, polished, readable, and kinetic**—like a Mars colony made into an arcade cabinet for kids.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about gameplay, audience, educational goals, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to this aesthetic direction:
- **Purpose**: The interface helps children learn multiplication tables through fast arcade play, clear feedback, and delightful sci-fi presentation.
- **Tone**: Retro-futuristic, kid-accessible, low-poly arcade command center. It should feel exciting, optimistic, and tactile rather than gritty, realistic, or overly flashy.
- **Constraints**: Must remain highly legible for children, responsive across devices, and performant enough for game-like UI transitions.
- **Differentiation**: The unforgettable quality is the feeling that the player is operating a tiny modular space base on a dusty orange planet while solving times tables in real time.

**CRITICAL**: Keep the experience visually exciting without losing clarity. Educational game UI lives or dies by readability. Every flourish must support comprehension, pace, and reward.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear low-poly sci-fi point-of-view
- Meticulously refined for clarity, delight, and fast interaction

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Do **not** change the fonts. Use **Orbitron** for display moments, scores, labels, HUD headings, countdowns, and major numeric emphasis. Use **Exo 2** for supporting UI, buttons, helper text, instructions, and educational copy. Typography should feel crisp, engineered, and arcade-ready.
- **Color & Theme**: Base the palette on deep space navy backgrounds, soft white and cool gray structures, and saturated construction-orange accents. Dusty terra-cotta terrain tones can be used for surfaces, cards, level panels, and decorative shapes. Avoid random neon overload; orange is the hero accent, not rainbow chaos.
- **Forms & Surfaces**: Favor beveled cards, angled corners, hexagonal motifs, modular panels, launch-bay framing, and chunky toy-like silhouettes. UI should echo the geometry of hangars, rockets, tracks, crates, and sci-fi base components.
- **Motion**: Use motion that feels mechanical and arcade-like: panel slides, score ticks, targeting pulses, scan sweeps, answer lock-ons, hover lifts, docking animations, and soft thrust/glow feedback. Avoid gooey or organic motion styles.
- **Spatial Composition**: Think in layers like an isometric game map. Build interfaces with command panels, status rails, mission cards, launch pads, radar widgets, and score modules. Layouts can feel structured and modular rather than generic website sections.
- **Backgrounds & Visual Details**: Use deep atmospheric gradients, subtle starfields, grid lines, radar rings, scanlines, dust haze, soft shadows, and restrained highlights. Decorative details should feel like environmental UI from a clean children's sci-fi strategy game.
- **Game Feel**: Buttons should feel like arcade controls. Answer states should feel immediate and rewarding. Correct answers may trigger energy pulses, badge flashes, ship boosts, or launch feedback. Incorrect answers should respond clearly without feeling punishing.
- **Educational Clarity**: Multiplication problems, answer choices, streaks, and progress indicators must always be easy to parse at a glance. Prioritize hierarchy, contrast, and spacing over visual cleverness.

## Visual Direction Rules

- Design like a **space learning arcade**, not a generic SaaS dashboard.
- Lean into **low-poly isometric sci-fi base language**.
- Keep the interface **friendly for kids** without becoming babyish.
- Make the world feel **constructed from modular colony parts**.
- Use **orange as the energetic gameplay accent** and dark navy as the atmospheric backdrop.
- Favor **chunky precision** over thin, delicate styling.
- Treat menus, HUDs, cards, and buttons like parts of the same base ecosystem.

## Avoid

- Generic app-store game UI clichés with random neon everywhere
- Military sci-fi or dystopian realism
- Overcomplicated effects that reduce readability
- Soft corporate SaaS layouts disguised as a game
- Cartoon aesthetics that clash with the clean low-poly model language
- Changing away from **Exo 2** and **Orbitron**

## Implementation Notes

When generating interfaces, prefer:
- CSS variables for a tightly controlled theme system
- Reusable HUD and panel components
- Strong hover/focus/pressed states for controls
- Clear score, streak, timer, and answer-state feedback
- Lightweight visual effects that still feel premium
- Accessible contrast and scalable text for younger players

## Core Aesthetic Keywords

Low-poly isometric sci-fi, modular colony, Mars outpost arcade, kids space academy, orange-and-white engineering, toy-like industrial shapes, command console HUD, multiplication blaster, launch pad UI, polished educational arcade.

Remember: this should feel like a **beautiful little space base built for learning times tables at full arcade speed**. Not sterile. Not generic. Not chaotic. Think: *mission control for kids, with a rocket parked outside and multiplication problems incoming fast.*
